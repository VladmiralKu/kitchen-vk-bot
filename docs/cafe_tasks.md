# Раздел "Задачи кафе"

Статус: запланировано после запуска кухонного MVP.

## Идея

`Задачи кафе` - отдельный раздел бота для внутренних задач, которые не являются заказами кухни и не относятся к стоп-листу.

Пример: "проверить морозилку", "помыть витрину", "позвонить поставщику", "пересчитать лимонад".

## Права

- Создавать задачи могут только `admin`.
- Смотреть активные задачи могут все активные сотрудники: `admin`, `waiter`, `cook`.
- Отмечать результат задачи могут все активные сотрудники.
- Удалять или править задачу после создания может только `admin`.

## Навигация

Главная кнопка: `Задачи кафе`.

Внутри раздела:

- `Задачи` - список активных задач.
- `Выполненные` - список закрытых задач, включая выполненные и заваленные/отменённые.

## Создание задачи

Администратор заходит в `Задачи кафе` и пишет свободным текстом.

Бот создаёт задачу и переносит её в подраздел `Задачи`.

Пример:

```text
Проверить морозилку до 18:00
```

Ответ бота:

```text
Задача создана:
Проверить морозилку до 18:00
```

После создания задача видна в списке активных задач как кнопка.

## Активные задачи

В подразделе `Задачи` бот показывает активные задачи кнопками.

Пример:

```text
Задачи кафе

[Проверить морозилку до 18:00]
[Помыть витрину]
[Пересчитать лимонад]
```

При нажатии на задачу бот спрашивает:

```text
Что с задачей?
Проверить морозилку до 18:00

[Выполнили]
[Запороли/отменили]
[Назад]
```

## Закрытие задачи

Если сотрудник нажал `Выполнили`, задача получает статус `done`.

Если сотрудник нажал `Запороли/отменили`, задача получает статус `failed`.

В обоих случаях сохраняется:

- кто закрыл задачу;
- когда закрыл;
- какой результат выбрал.

## Выполненные

В подразделе `Выполненные` показываются закрытые задачи:

```text
Выполненные задачи

[выполнено] Проверить морозилку до 18:00
Закрыл: Иван, 17:42

[запороли/отменили] Помыть витрину
Закрыл: Сергей, 18:10
```

## Статусы

- `open` - активная задача.
- `done` - выполнена.
- `failed` - запороли/отменили.
- `deleted` - скрыта администратором, история сохраняется.

## Таблица базы

Планируемая таблица: `cafe_tasks`.

- `id UUID PK`
- `text TEXT NOT NULL`
- `status TEXT CHECK status IN ('open', 'done', 'failed', 'deleted')`
- `created_by UUID FK users.id`
- `created_at TIMESTAMPTZ`
- `closed_by UUID NULL FK users.id`
- `closed_at TIMESTAMPTZ NULL`
- `close_comment TEXT NULL`
- `deleted_at TIMESTAMPTZ NULL`

## Callback payload

Открыть задачу:

```json
{
  "action": "open_cafe_task",
  "task_id": "uuid"
}
```

Закрыть задачу:

```json
{
  "action": "close_cafe_task",
  "task_id": "uuid",
  "result": "done"
}
```

```json
{
  "action": "close_cafe_task",
  "task_id": "uuid",
  "result": "failed"
}
```

## Критерии готовности

1. В меню есть раздел `Задачи кафе`.
2. Только админ может создать задачу свободным текстом внутри раздела.
3. Активная задача появляется в подразделе `Задачи` как кнопка.
4. Нажатие на задачу открывает выбор `Выполнили` или `Запороли/отменили`.
5. После выбора задача исчезает из активных и появляется в `Выполненные`.
6. В закрытой задаче видно, кто и когда её закрыл.
7. Удаление задачи мягкое: история не теряется.
